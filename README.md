# NoteTube AI

Turn any YouTube lecture into a full academic study pack — an executive summary,
comprehensive chapter notes, key takeaways, 20 MCQs with an answer key, and 10
technical interview questions — generated by Gemini 2.5 Flash and exportable as a
styled Word document.

```
notetube-ai/
├── backend/
│   ├── app.py            Flask app: /api/generate, /api/download/<file>, /api/health
│   ├── transcript.py     YouTube URL parsing + transcript fetching
│   ├── ai.py              Gemini integration + prompt design
│   ├── doc_generator.py  Markdown -> styled .docx export
│   └── .env.example      Environment variable template
├── frontend/
│   ├── index.html
│   ├── style.css
│   └── script.js
├── requirements.txt
└── README.md
```

https://notetube-ai-zeta.vercel.app/


## 1. Prerequisites

- Python 3.11+
- A Gemini API key from [Google AI Studio](https://aistudio.google.com/app/apikey)
- Any static file server for the frontend (VS Code's "Live Server" extension works well)

## 2. Backend setup

```bash
cd notetube-ai
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

pip install -r requirements.txt

cp backend/.env.example backend/.env
# then open backend/.env and set GEMINI_API_KEY to your real key
```

Run the server:

```bash
cd backend
python app.py
```

The API will be live at `http://localhost:5000`. Confirm it's healthy:

```bash
curl http://localhost:5000/api/health
```

## 3. Frontend setup

The frontend is plain HTML/CSS/JS with no build step. Open `frontend/index.html`
with a local static server (opening the file directly also works, but a server
avoids CORS quirks with some browsers).

If your backend isn't running on `http://localhost:5000`, set the base URL before
`script.js` loads by adding this to `index.html`, right above the `<script src="script.js">`
tag:

```html
<script>window.NOTETUBE_API_BASE = "https://your-deployed-backend.onrender.com";</script>
```

## 4. Using the app

1. Paste a YouTube URL with captions enabled into the input box.
2. Click **Generate Notes** and wait — longer videos take longer to process.
3. Browse the generated **Study Notes**, **Quizzes & MCQs**, and **Interview Prep** tabs.
4. Use **Copy Notes** to copy the raw Markdown, or go to the **Downloads** tab for the
   formatted Word document.

## 5. Deployment

### Backend → Render

1. Push this repository to GitHub.
2. In Render, create a **New Web Service** pointing at the repo, with root directory `backend`.
3. Build command: `pip install -r ../requirements.txt`
4. Start command: `gunicorn app:app` (add `gunicorn` to `requirements.txt` for production, or use `python app.py` for a quick test deploy)
5. Add environment variables from `backend/.env.example` in Render's dashboard —
   at minimum `GEMINI_API_KEY`, and set `CORS_ALLOWED_ORIGINS` to your deployed
   frontend's URL.

### Frontend → Vercel

1. In Vercel, create a **New Project** pointing at this repo, with root directory `frontend`.
2. Framework preset: **Other** (static site — no build command needed).
3. After deploying, add the `window.NOTETUBE_API_BASE` snippet from step 3 above,
   pointing at your Render backend URL, and redeploy.

## 6. Known limitations

- Videos without captions/transcripts cannot currently be processed (no
  speech-to-text fallback is included in this build).
- Extremely long videos (multi-hour) are chunked automatically before being sent
  to Gemini, but very long transcripts still take noticeably longer to process.
- Generated files are stored temporarily in `backend/temp_files/` and are not
  automatically purged in this build — consider adding a cleanup job (e.g. a
  cron task or scheduled Render job) using `FILE_CLEANUP_MAX_AGE_SECONDS` from
  `.env.example` before running this in production.

## 7. A note on scope

The build in this repo covers **Word (.docx) export only**. If you also need PDF
export, `reportlab` and a `pdf_generator.py` module can be added following the same
Markdown-to-document mapping pattern used in `doc_generator.py`.
